Création de nouveaux programmes

Des programmes peuvent être créés suivant les besoins de l’équipe ou des partenaires. Pour créer un programme, il faut sélectionner l’option correspondante dans le menu déroulant au-dessus de la liste des programmes. Le formulaire suivant apparaît :

arrow_forward Programme : Nom du programme

arrow_forward ID du programme : Identifiant du programme.

arrow_forward Modalité : En ligne ou en présentiel

arrow_forward Description : Description du cours (facultatif)

arrow_forward Contenu : Contenu du cours (facultatif)

arrow_forward Date de création : À laisser de préférence à la date du jour de création

arrow_forward Date de début et de fin : Début et fin du programme

arrow_forward Durée et coût total : Facultatif

En dessous de cette première partie, se trouve un tableau pour sélectionner le centre du programme.

En effet, certains programmes n’existent que pour certains centres. Il serait malvenu que tous les centres puissent les voir. Pour choisir les centres dans lesquels le programme sera disponible, il suffit de cliquer sur le « + » pour ajouter. Une fois cette étape accomplie, il faut sélectionner les cours à associer au programme. Dans notre cas, on configurera uniquement des cours en ligne.

On sélectionne les cours dans la liste déroulante et on enregistre pour ajouter le cours. Un programme peut tout à fait contenir plusieurs cours (cf programme A1-6 unités).

Liste des cours en ligne et présentiels

Les listes des cours en ligne et présentiels sont tirées de Moodle. Dans notre cas, nous n’avons pas de cours présentiels. Cette liste est à mettre à jour quand on crée un nouveau cours sur Moodle. Chaque cours possède sa « fiche » avec ses éléments.

Illustration 7: Fiche du cours A1-6 unités

L’id du cours est utilisé dans l’upload automatique de tuteurs sur une session (voir §3.5). Il figure dans le fichier CSV sous le champ COURS. IMPORTANT : Lors de l’ajout d’un tuteur sur la plateforme, il faut aussi mettre à jour la liste des cours pour que le tuteur apparaisse bien dans la liste des tuteurs disponibles pour le cours. Si le tuteur n’apparaît pas dans la liste lors de la création d’une session, c’est parce que sur la fiche du cours, il n’est pas dans la liste des tuteurs disponibles.

Sessions

La création de sessions est entièrement traitée dans le guide du gestionnaire. Chaque centre partenaire est autonome dans la gestion de ses sessions.

Télécharger les sessions

Le fichier CSV demandé par cette fonction nous permet de créer des nouvelles sessions automatiquement. Les sessions sont créées sur des programmes déjà définis.

La liste des champs à remplir suit le formulaire manuel de créations de sessions.

Cependant, il faut faire attention à deux choses :

- ID : Quand il s’agit d’une nouvelle session, l’ID a les valeurs new, NEW, 0 ou vide. Aussi étrange que cela puisse paraître, ISYC nous affirme que nous pouvons laisser ce champ vide.

- CODE DE LA SESSION ou NOM DE GROUPE : Il faut remplir un de ces deux champs afin de d’identifier la session dans le cas d’une modification.

- PROGRAMME : Ce champ fait référence au nom du programme et non à l’ID du programme (cf §3.1)

Télécharger les tuteurs

Malgré un titre trompeur, cette option nous permet d’assigner automatiquement des tuteurs à des sessions déjà créées.

La liste des champs à remplir est la suivante :

- METHODE : La procédure pour remplir ce champ est expliquée sur la plateforme

- UTILISATEUR : Nom d’utilisateur (adresse mail)

- ROLE : Le rôle du tuteur. Le chiffre 3 correspond au rôle d’éditeur. Le chiffre 4 à celui de tuteur.

- SESSION : Nom de la session

- COURS : Id du cours
